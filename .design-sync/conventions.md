# SIMON THINKS — Design System sync

This repository is a **mirror** of the SIMON THINKS design-system project authored
in the design tool. The design-tool project is the **source of truth**.

## What lives here

| Path | What it is |
|------|------------|
| `styles.css` | Single global-CSS entry point (consumers link this). `@import`s all tokens. |
| `tokens/` | Design tokens — colors, typography, spacing, effects, fonts, **icons** (578-glyph Pixel Icon font). |
| `assets/` | Logos, symbol marks, reference imagery, and the icon webfont (`assets/icons/`). |
| `components/` | React components (`.jsx` + `.d.ts` + `.prompt.md`) grouped by family, each with a `*.card.html` showcase. |
| `guidelines/` | Brand, color, type, spacing and application showcase cards (`*.card.html`). |
| `integrations/` | Tailwind v3 preset, Tailwind v4 `@theme`, framework-agnostic `.stk-*` classes. |
| `_ds_bundle.js` | **Generated** — compiled component bundle. Read components via `window.SIMONTHINKSDesignSystem_ae45ac`. |
| `_ds_manifest.json` | **Generated** — component / card / token manifest. |

## Consuming the components

```html
<script src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>
<link rel="stylesheet" href="styles.css">
<script src="_ds_bundle.js"></script>
<script>
  const { Button, Icon, BarChart } = window.SIMONTHINKSDesignSystem_ae45ac;
</script>
```

Plain HTML icon usage (no React): `<i class="hn hn-search"></i>` — see `tokens/icons.css`.

## Updating (keeping in sync)

1. Edit in the **design-tool project** (source of truth).
2. Re-export / push the changed files here.
3. CI (`.github/workflows/validate.yml`) validates integrity on every push.

> ⚠️ Do **not** hand-edit `_ds_bundle.js` or `_ds_manifest.json` — they are
> regenerated by the design tool and will be overwritten on the next sync.
